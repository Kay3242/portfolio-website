This interactive chatbot-style page was built to simulate a conversation with the site’s developer, providing users with insights into the website’s structure, styling, and functionality. It serves as both a developer diary and an educational tool, styled to fit seamlessly within the Among Us aesthetic.

The UI was styled to resemble a messaging interface using custom CSS for rounded buttons, gradient backgrounds, and a modular layout. The visual design aligns with the rest of the site by using the Among Us-inspired color palette, typography, and avatar art, creating a consistent user experience.

Each question button on the left is hard-coded with an event listener in JavaScript. When clicked, the corresponding answer text is dynamically injected into the right-hand dialogue box using innerHTML. This modular structure made it easy to update or expand the number of questions without rewriting core logic.

The Javascript logic handles active states and question tracking using conditional statements and class toggling. It ensures that only one message is visible at a time, mimicking a clean chatbot reply flow without relying on external libraries or back-end frameworks.

All elements were coded using semantic HTML and styled with scalable rem units and flexbox. This allows the chatbot interface to remain fully functional and readable across desktop and mobile viewports. Button focus states and hover effects were also added for accessibility and visual feedback.
